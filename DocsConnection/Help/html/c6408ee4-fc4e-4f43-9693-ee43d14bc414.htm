DataAccessLibrary class documentation

The DatabaseConnection class provides an abstract base for managing database connections and executing commands. It encapsulates common functionalities such as opening and closing connections, retrieving, and updating data.

Initialization

The DatabaseConnection constructor initializes a new instance, setting up the connection string and SQL connection. It fetches the connection string from the application's configuration.

Data Retrieval

Methods like RetrieveAllDataFromTable, RetrieveDataUsingQuery, and RetrieveDataUsingCommand allow for fetching data from the database in various ways, supporting both simple table queries and complex SQL commands.

  • RetrieveAllDataFromTable(tableName) - Retrieves all data from a specified table.

  • RetrieveDataUsingQuery(sqlQuery) - Executes a specified SQL query and returns the result.

  • RetrieveDataUsingQuery(sqlQuery, tableName) - Similar to the above but structures the DataSet with a specified table name.

Data Modification

Methods like UpdateData and ExecuteSqlNonQuery facilitate modifying data within the database, supporting updates and executing non-query SQL commands.

  • UpdateData(sqlQuery, dataSetToUpdate) - Updates data in the database based on the provided DataSet.

  • ExecuteSqlNonQuery(sqlQuery) - Executes an SQL non-query command like INSERT, UPDATE, or DELETE.

Advanced Usage

The DatabaseConnection class also supports advanced scenarios like generating query commands dynamically and executing commands within transactions.

  • GenerateQuery(tableName, fieldValues) - Dynamically creates a SQL query command based on table name and field values.

  • ExecuteTransaction(command) - Executes a SQL command within a transaction, providing rollback capabilities in case of errors.
